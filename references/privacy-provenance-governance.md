# Privacy, Provenance, And Reuse Governance

Use this for user-uploaded assets, client materials, private scripts, reference images/videos, brand work, and any reusable production asset.

## Rule

Do not treat every user asset as freely reusable inspiration. Track source, permission, and reuse boundary.

## Provenance Fields

```markdown
# Provenance Lock

| Asset | Source | Owner | Permission | Reuse Boundary | Public/Private | Notes |
|---|---|---|---|---|---|---|
```

## Asset Classes

| Class | Handling |
|---|---|
| User private photo | preserve only for this project unless user says reusable |
| Client logo/product | use within brief; avoid unauthorized claims |
| Reference film/video | extract craft only, do not copy protected expression |
| Stock/online image | note source; avoid assuming commercial clearance |
| Generated asset | track prompt/version/model; can reuse if project-approved |
| Voice/music | require rights clarity before public/commercial delivery |

## Public Release Gate

Before public or commercial delivery, check:

- no private face/voice without permission
- no client confidential detail exposed in prompt files if sharing externally
- no unlicensed logo/music/lyrics/footage treated as owned
- no hidden source references that cannot be disclosed in contest/client review
- no prompt package accidentally contains private paths, internal notes, or user secrets

## Final Prompt Rule

The final prompt package may include asset codes and reference duties, but should not expose private file paths or unnecessary personal information. Use codes such as `USER_CHAR_A_REF` instead of raw private filenames when sharing externally.
